# 上线准备中心规格说明

## 背景

本地 Sub2API 已能启动并通过健康检查，但距离公网收费上线还缺少上游账号、代理、渠道、支付服务商、套餐、备份与监控收口。用户希望在部署前先把后台功能补齐，后续通过后台逐项完成上线准备。

## 目标

- 在后台新增“上线准备中心”，直接回答“下一步应该干啥”。
- 后端提供 `/api/v1/admin/launch-readiness`，输出结构化检查报告。
- 前端提供 `/admin/launch-readiness` 页面，展示整体状态、完成度、分区检查和跳转动作。
- 检查项覆盖公网部署、系统初始化、Claude/Codex 网关、官方支付宝收费、备份与运维。
- 所有新增用户可见文案使用中文，并保留英文 i18n 兜底。

## 范围

- 包含：只读检查报告、后台导航入口、页面展示、动作跳转、测试和规则文档。
- 不包含：自动导入真实上游账号、自动申请支付宝、自动购买或配置 VPS、真实生产密钥生成。

## 检查口径

- 阻断项：缺少 HTTPS 前端地址、管理员、TOTP 固定密钥、活跃账号、代理、分组、渠道、API Key、支付宝服务商、可售套餐、部署/备份脚本。
- 警告项：未开启后台模式、公开注册未关闭、Ops 监控/告警未收口、微信支付首版仍可见。
- 通过项：所有阻断和警告均完成后，整体状态为可准备上线。

## 关键路径

1. 读取部署文件是否存在。
2. 读取系统设置和支付设置。
3. 统计管理员、分组、账号、代理、渠道、API Key、支付宝服务商、套餐和告警规则。
4. 聚合为分区报告。
5. 前端渲染报告，并给每个可操作检查项提供后台跳转。
